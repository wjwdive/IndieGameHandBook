# 快速参考指南

本章节是独立游戏开发的快速参考手册，汇集了常用命令、快捷键、配置示例和实用技巧。无论是日常开发中的快速查询，还是解决特定问题的参考，这里都能提供即时的帮助。

**学习目标**：
- 掌握常用开发工具的核心快捷键和命令
- 快速查找常见问题的解决方案
- 了解最佳实践配置示例
- 建立个人快速参考笔记系统

本指南按工具和主题组织，每个部分包含：
1. **核心快捷键** - 提高效率的关键组合

## 2、版本控制快速参考

### 2.1 Git 快速参考

#### 基础工作流
```bash
# 初始化仓库
git init
git remote add origin <仓库URL>

# 克隆仓库
git clone <仓库URL>

# 查看状态
git status

# 添加文件
git add <文件名>          # 添加特定文件
git add .                # 添加所有更改

# 提交更改
git commit -m "提交信息"

# 推送更改
git push origin <分支名>

# 拉取更新
git pull origin <分支名>
```

#### 分支管理
```bash
# 创建分支
git branch <分支名>
git checkout -b <分支名>  # 创建并切换

# 切换分支
git checkout <分支名>

# 合并分支
git merge <分支名>

# 删除分支
git branch -d <分支名>    # 安全删除
git branch -D <分支名>    # 强制删除
```

#### 解决冲突
```bash
# 查看冲突文件
git status

# 手动解决冲突后
git add <冲突文件>
git commit -m "解决冲突"
```

#### Git LFS 大文件管理
```bash
# 安装 Git LFS
git lfs install

# 跟踪大文件类型
git lfs track "*.psd"
git lfs track "*.wav"
git lfs track "*.fbx"

# 查看跟踪的文件类型
git lfs track
```

#### 协程模式
```csharp
// 启动协程
StartCoroutine(MyCoroutine());

IEnumerator MyCoroutine() {
    yield return null;           // 等待下一帧
    yield return new WaitForSeconds(1f); // 等待1秒
    yield return new WaitForEndOfFrame(); // 等待帧结束
    yield return new WaitUntil(() => condition); // 等待条件满足
    yield return new WaitWhile(() => condition); // 等待条件为真
}
```

#### 属性访问器模式
```csharp

}

public string PlayerName { get; set; }
```

### 3.2 GDScript (Godot) 快速参考

#### 信号和事件
```gdscript
# 定义和发射信号
signal health_changed(old_value, new_value)

func take_damage(amount: int):
    var old_health = health
    health -= amount
    emit_signal("health_changed", old_health, health)

# 连接信号（代码中）
func _ready():
    connect("health_changed", self, "_on_health_changed")

func _on_health_changed(old_val, new_val):
    print("血量变化: ", old_val, " -> ", new_val)
```

## 7、故障排除快速参考

### 7.1 常见编译错误解决方案

#### Unity 编译错误
```
CS0246: 找不到类型或命名空间名称
解决方案：检查命名空间引用，确保 using 语句正确

NullReferenceException: Object reference not set to an instance of an object
解决方案：检查变量是否初始化，使用 null 检查
```

#### Godot 错误
```
Parser Error: Expected ':' in type declaration
解决方案：检查变量声明语法 var name: Type = value

Node not found: NodeName
解决方案：检查节点路径是否正确，使用相对或绝对路径
```

### 7.2 性能问题快速诊断

#### 帧率低下
1. **检查 Draw Calls**：使用帧调试器查看绘制调用数量
2. **检查三角形数量**：确保模型多边形数量合理
3. **检查材质数量**：合并材质，减少切换
4. **检查脚本性能**：使用性能分析器找到热点代码

#### 内存泄漏
1. **检查对象引用**：确保适当释放不再使用的对象
2. **使用内存分析器**：定期检查内存使用情况
3. **检查资源加载**：确保正确卸载资源

### 7.3 平台特定问题

#### Android 构建问题
```
Failed to compile shaders
解决方案：检查着色器兼容性，使用更简单的着色器

APK太大
解决方案：使用纹理压缩，移除未使用资源
```

#### iOS 构建问题
```
Code signing errors
解决方案：检查证书和描述文件配置

Missing icons
解决方案：确保包含所有所需尺寸的图标
```
