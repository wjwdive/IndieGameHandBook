# 游戏编程基础

## 概述

本章将介绍游戏编程的核心概念和基础原理。无论你使用哪种游戏引擎或编程语言，理解这些基础知识都将帮助你更好地掌握游戏开发技术，写出更高效、更稳定的游戏代码。我们将从最基础的游戏循环开始，逐步深入到游戏状态管理、输入处理、时间系统等关键概念。

**学习目标**：
- 理解游戏循环（Game Loop）的工作原理和重要性
- 掌握游戏状态管理和状态转换的基本方法
- 学会处理用户输入和事件系统
- 理解时间系统和帧率控制的概念
- 掌握游戏开发中常用的数学基础知识
- 了解场景图（Scene Graph）的基本概念

## 1、游戏循环：游戏的心脏

### 1.1 什么是游戏循环

游戏循环（Game Loop）是游戏程序最核心的部分，它负责持续运行游戏逻辑，保持游戏世界"活着"。可以把它想象成游戏的心脏——不断地跳动，驱动着整个游戏系统。

一个基本的游戏循环包含三个主要阶段：
1. **处理输入（Process Input）**：读取玩家操作（键盘、鼠标、手柄等）
2. **更新游戏状态（Update Game State）**：根据输入和时间推进游戏世界
3. **渲染输出（Render Output）**：将游戏状态绘制到屏幕上

### 1.2 游戏循环的实现

以下是一个最简单的游戏循环示例（伪代码）：

```python
def game_loop():
    initialize_game()  # 初始化游戏

    while game_is_running:  # 主循环
        # 1. 处理输入
        inputs = process_input()

        # 2. 更新游戏状态
        update_game(inputs)

        # 3. 渲染输出
        render_game()

        # 控制帧率
        limit_frame_rate()

    cleanup_game()  # 游戏结束清理
```

> **注意**：实际游戏中，游戏循环更加复杂，需要考虑时间插值、多线程渲染、网络同步等高级特性。但所有游戏循环都基于这个基本模式。

### 1.3 固定时间步长 vs 可变时间步长

游戏循环的时间处理有两种主要方式：

| 类型 | 工作原理 | 优点 | 缺点 | 适用场景 |
|------|---------|------|------|---------|
| **固定时间步长** | 每次更新使用固定的时间增量（如1/60秒） | 物理模拟稳定，可预测性高 | 可能浪费CPU资源，帧率受限 | 物理密集型游戏、多人游戏 |
| **可变时间步长** | 更新使用实际经过的时间 | 充分利用硬件性能，帧率灵活 | 物理模拟可能不稳定，时间相关逻辑复杂 | 图形密集型游戏、单机游戏 |

大多数现代游戏引擎采用**混合方法**：使用固定时间步长更新物理和游戏逻辑，使用可变时间步长进行渲染。

## 2、游戏状态管理

### 2.1 游戏状态的概念

游戏状态（Game State）指在某一时刻游戏世界的完整描述，包括：
- 玩家位置、生命值、装备等
- 敌人位置、行为状态
- 物品位置、状态
- 游戏进度、分数、时间等

### 2.2 状态转换与状态机

游戏通常有多个状态，如主菜单、游戏进行中、暂停、游戏结束等。管理这些状态转换的常用工具是**有限状态机（Finite State Machine, FSM）**。

```python
class GameStateMachine:
    def __init__(self):
        self.current_state = "MAIN_MENU"
        self.states = {
            "MAIN_MENU": MainMenuState(),
            "PLAYING": PlayingState(),
            "PAUSED": PausedState(),
            "GAME_OVER": GameOverState()
        }

    def change_state(self, new_state):
        # 退出当前状态
        self.states[self.current_state].exit()

        # 进入新状态
        self.current_state = new_state
        self.states[new_state].enter()

    def update(self, delta_time):
        self.states[self.current_state].update(delta_time)

    def render(self):
        self.states[self.current_state].render()
```

### 2.3 数据驱动设计

优秀的游戏状态管理应该**数据驱动**，将游戏数据与逻辑分离：

```json
{
  "player": {
    "position": {"x": 100, "y": 200},
    "health": 100,
    "inventory": ["sword", "potion"]
  },
  "enemies": [
    {"type": "goblin", "position": {"x": 300, "y": 150}, "health": 50}
  ],
  "level": {
    "name": "Forest Dungeon",
    "completed": false
  }
}
```

这样设计的好处：
- 便于保存/加载游戏（存档功能）
- 便于调试和测试
- 便于内容创建（关卡编辑器）
- 逻辑与数据分离，提高代码可维护性

## 3、输入处理系统

### 3.1 输入设备抽象

现代游戏需要支持多种输入设备：键盘、鼠标、手柄、触摸屏等。良好的输入系统应该将这些设备**抽象化**，提供统一的接口：

```python
class InputSystem:
    def is_pressed(self, action):

        key_mapping = self.config.get_mapping(action)

        # 检查所有映射的输入源
        for device, button in key_mapping:
            if self.devices[device].is_pressed(button):
                return True
        return False

    def get_axis(self, axis_name):
        # 处理模拟输入（摇杆、鼠标移动等）
        # 返回-1.0到1.0之间的值
        pass
```

### 3.2 输入配置与重映射

允许玩家自定义按键是专业游戏的标配。实现方法：
1. 使用**输入动作（Input Actions）**而非直接按键检测
2. 提供可配置的按键映射
3. 支持多种输入设备自动切换

```python
# 不好的做法：直接检测特定按键
if keyboard.is_pressed("SPACE"):
    player.jump()

# 好的做法：通过动作抽象
if input_system.is_pressed("JUMP"):
    player.jump()

# 配置文件中定义映射
{
  "JUMP": [
    {"device": "keyboard", "button": "SPACE"},
    {"device": "gamepad", "button": "A"},
    {"device": "gamepad", "button": "X"}
  ]
}
```

### 3.3 输入缓冲与组合输入

对于动作游戏，输入缓冲和组合输入非常重要：

- **输入缓冲**：在动作不可用时暂时存储输入，在动作可用时立即执行
- **组合输入**：检测特定顺序的输入组合（如"下+前+拳"）

```python
class InputBuffer:
    def __init__(self, buffer_time=0.2):
        self.buffer = {}
        self.buffer_time = buffer_time

    def record_input(self, action, timestamp):
        self.buffer[action] = timestamp

    def check_buffered(self, action, current_time):
        if action in self.buffer:
            # 检查输入是否仍在缓冲时间内
            if current_time - self.buffer[action] <= self.buffer_time:
                del self.buffer[action]  # 消耗缓冲的输入
                return True
        return False
```

## 4、时间系统与帧率控制

### 4.1 Delta Time（时间增量）

`delta_time`是游戏编程中最重要的概念之一，表示上一帧到当前帧经过的时间（通常以秒为单位）：

```python
current_time = get_current_time()
delta_time = current_time - last_frame_time
last_frame_time = current_time

# 使用delta_time使运动与时间而非帧率相关
player.position.x += player.speed * delta_time
```

> **重要**：永远使用`delta_time`来计算基于时间的运动，否则游戏在不同帧率下的运行速度会不同。

### 4.2 帧率控制策略

| 策略 | 实现方法 | 适用场景 |
|------|---------|---------|
| **垂直同步（VSync）** | 等待显示器刷新，消除画面撕裂 | 大多数单机游戏 |
| **固定帧率** | 精确控制每帧时间，稳定性能表现 | 竞技游戏、模拟游戏 |
| **无上限帧率** | 尽可能快地运行，最大化性能 | 基准测试、电竞 |
| **动态分辨率** | 根据帧率调整渲染分辨率 | 性能有限的平台（如移动设备） |

### 4.3 时间缩放与慢动作效果

时间系统还可以实现特殊效果：

```python
class TimeManager:
    def __init__(self):
        self.time_scale = 1.0  # 正常时间流速
        self.paused = False

    def get_delta_time(self):
        base_delta = get_raw_delta_time()

        if self.paused:
            return 0.0

        return base_delta * self.time_scale

    def set_slow_motion(self, scale=0.3, duration=2.0):
        # 设置慢动作效果
        self.time_scale = scale
        # 可以添加定时器在duration后恢复

    def set_pause(self, paused=True):
        self.paused = paused
```

## 5、游戏数学基础

### 5.1 向量数学

向量是游戏开发中最常用的数学工具，用于表示位置、速度、方向等：

```python
class Vector2:
    def __init__(self, x=0.0, y=0.0):
        self.x = x
        self.y = y

    def length(self):
        return math.sqrt(self.x * self.x + self.y * self.y)

    def normalize(self):
        length = self.length()
        if length > 0:
            return Vector2(self.x / length, self.y / length)
        return Vector2(0, 0)

    def dot(self, other):
        return self.x * other.x + self.y * other.y

    # 向量加法、减法、数乘等...
```

常用向量运算：
- **标准化**：获取方向（长度为1的向量）
- **点积**：计算夹角、投影
- **叉积**：计算法向量、确定方向
- **线性插值（Lerp）**：平滑过渡

### 5.2 矩阵与变换

矩阵用于表示复杂的变换：

| 变换类型 | 矩阵形式 | 游戏中的应用 |
|---------|---------|------------|
| **平移** | ![\begin{bmatrix} 1 & 0 & t_x \\ 0 & 1 & t_y \\ 0 & 0 & 1 \end{bmatrix}](https://latex.codecogs.com/svg.latex?%5Cbegin%7Bbmatrix%7D%201%20%260%20%26t_x%20%5C%5C%200%20%261%20%26t_y%20%5C%5C%200%20%260%20%261%20%5Cend%7Bbmatrix%7D) | 移动物体位置 |
| **旋转** | ![\begin{bmatrix} \cos\theta & -\sin\theta & 0 \\ \sin\theta & \cos\theta & 0 \\ 0 & 0 & 1 \end{bmatrix}](https://latex.codecogs.com/svg.latex?%5Cbegin%7Bbmatrix%7D%20%5Ccos%5Ctheta%20%26-%5Csin%5Ctheta%20%260%20%5C%5C%20%5Csin%5Ctheta%20%26%5Ccos%5Ctheta%20%260%20%5C%5C%200%20%260%20%261%20%5Cend%7Bbmatrix%7D) | 旋转物体方向 |
| **缩放** | ![\begin{bmatrix} s_x & 0 & 0 \\ 0 & s_y & 0 \\ 0 & 0 & 1 \end{bmatrix}](https://latex.codecogs.com/svg.latex?%5Cbegin%7Bbmatrix%7D%20s_x%20%260%20%260%20%5C%5C%200%20%26s_y%20%260%20%5C%5C%200%20%260%20%261%20%5Cend%7Bbmatrix%7D) | 改变物体大小 |

### 5.3 插值与缓动函数

插值用于创建平滑动画：

```python
def lerp(start, end, t):
    """线性插值"""
    return start + (end - start) * t

def ease_in_out(t):
    """缓入缓出函数"""
    return t * t * (3.0 - 2.0 * t)

def smooth_step(t):
    """更平滑的过渡"""
    return t * t * t * (t * (t * 6 - 15) + 10)

# 使用示例
current_position = lerp(start_position, target_position, ease_in_out(progress))
```

## 6、场景图与游戏对象系统

### 6.1 场景图概念

### 6.2 组件系统

现代游戏引擎多采用组件系统（Component System）：

```python
class GameObject:
    def __init__(self, name):
        self.name = name
        self.components = {}
        self.transform = Transform()
        self.children = []
        self.parent = None

    def add_component(self, component_type):
        component = component_type(self)
        self.components[component_type] = component
        return component

    def get_component(self, component_type):
        return self.components.get(component_type)

    def update(self, delta_time):
        for component in self.components.values():
            component.update(delta_time)

        for child in self.children:
            child.update(delta_time)

# 组件示例
class SpriteRenderer:
    def __init__(self, game_object):
        self.game_object = game_object
        self.texture = None
        self.color = (255, 255, 255)

    def update(self, delta_time):
        # 渲染逻辑
        pass

class PhysicsBody:
    def __init__(self, game_object):
        self.game_object = game_object
        self.velocity = Vector2(0, 0)
        self.mass = 1.0

    def update(self, delta_time):
        # 物理更新
        self.game_object.transform.position += self.velocity * delta_time
```

### 6.3 实体-组件-系统（ECS）架构

ECS是更高级的架构模式，提供更好的性能：

- **实体（Entity）**：只有ID的容器，不包含数据或逻辑
- **组件（Component）**：纯数据，没有逻辑
- **系统（System）**：纯逻辑，处理具有特定组件组合的实体

```python
# ECS简化示例
class PositionComponent:
    def __init__(self, x=0.0, y=0.0):
        self.x = x
        self.y = y

class MovementSystem:
    def update(self, entities, delta_time):
        for entity in entities:
            if entity.has(PositionComponent) and entity.has(VelocityComponent):
                pos = entity.get(PositionComponent)
                vel = entity.get(VelocityComponent)
                pos.x += vel.x * delta_time
                pos.y += vel.y * delta_time
```
