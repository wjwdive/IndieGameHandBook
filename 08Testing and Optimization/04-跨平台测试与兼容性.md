# 跨平台测试与兼容性

## 概述

跨平台游戏开发是现代独立游戏的常态，但每个平台都有独特的硬件特性、操作系统限制和用户体验期望。系统化的跨平台测试确保游戏在所有目标平台上都能提供一致、高质量的体验。本章将深入探讨跨平台测试的策略、工具和实践，涵盖从PC到移动端、主机到云游戏的全平台兼容性保障。

**学习目标**：
- 理解不同游戏平台的硬件和软件特性差异
- 掌握跨平台兼容性测试的核心方法和工具
- 学习平台特定功能的测试策略（如手柄支持、触摸屏、平台API）
- 掌握多平台自动化测试的配置和执行
- 了解平台认证要求和合规性测试

## 1、平台特性与测试挑战

### 1.1 主要游戏平台对比

不同游戏平台在硬件能力、输入方式、用户期望和发布要求方面存在显著差异。

**PC平台（Windows/macOS/Linux）**：
- **硬件多样性**：广泛的CPU、GPU、内存配置
- **输入方式**：键盘鼠标为主，手柄为可选
- **分辨率范围**：从低分辨率笔记本到4K/8K多显示器
- **用户期望**：高性能、可自定义设置、Mod支持

**游戏主机（PlayStation/Xbox/Nintendo）**：
- **硬件统一性**：固定硬件配置，优化目标明确
- **输入标准化**：专用手柄，一致的输入映射
- **性能要求**：严格的帧率和分辨率标准
- **认证流程**：平台持有者要求的完整认证测试

**移动平台（iOS/Android）**：
- **硬件碎片化**：数千种不同设备配置
- **输入方式**：触摸屏为主，传感器（陀螺仪、加速度计）支持
- **性能限制**：能效优先，散热限制，电池寿命考虑
- **商店要求**：应用商店审核，内容分级，年龄限制

**新兴平台（云游戏/VR/AR）**：
- **云游戏**：网络延迟敏感性，流式传输质量
- **VR/AR**：运动追踪精度，眩晕预防，空间音频
- **Web游戏**：浏览器兼容性，WebGL性能，加载优化

### 1.2 跨平台兼容性核心问题

识别跨平台开发中的常见兼容性问题，有助于针对性测试。

**图形渲染兼容性**：
- **着色器语言差异**：HLSL vs GLSL vs Metal SL
- **纹理格式支持**：不同平台的压缩纹理格式（DXT, ETC2, PVRTC, ASTC）
- **渲染API差异**：DirectX vs OpenGL vs Vulkan vs Metal
- **后处理效果兼容性**：某些效果在特定硬件上可能不可用或性能差

**输入系统兼容性**：
- **手柄映射差异**：不同平台手柄的按钮布局和轴映射
- **触摸屏手势**：移动端特定手势（捏合缩放、滑动）
- **键盘布局**：国际键盘布局差异，特殊键位支持
- **辅助功能**：平台特定的辅助功能支持要求

**音频系统兼容性**：
- **音频格式支持**：不同平台的编解码器支持差异
- **3D音频API**：空间音频实现的平台差异
- **音频会话管理**：移动端音频会话中断处理

**系统集成兼容性**：
- **平台API集成**：成就、排行榜、云存档等平台服务
- **多任务处理**：后台运行、通知处理、电源管理
- **文件系统权限**：不同平台的沙盒限制和文件访问权限

```csharp
// 跨平台输入处理示例
using UnityEngine;

public class CrossPlatformInputHandler : MonoBehaviour
{
    // 平台特定的输入映射
    [System.Serializable]
    public class PlatformInputMapping
    {
        public RuntimePlatform platform;
        public string jumpButton = "Jump";
        public string attackButton = "Fire1";
        public KeyCode pauseKey = KeyCode.Escape;
        public string horizontalAxis = "Horizontal";
        public string verticalAxis = "Vertical";
    }

    public PlatformInputMapping[] platformMappings;
    private PlatformInputMapping currentMapping;

    void Start()
    {
        // 根据当前平台选择输入映射
        currentMapping = GetMappingForPlatform(Application.platform);
        Debug.Log($"使用输入映射: {currentMapping.platform}");
    }

    PlatformInputMapping GetMappingForPlatform(RuntimePlatform platform)
    {
        foreach (var mapping in platformMappings)
        {
            if (mapping.platform == platform)
                return mapping;
        }

        // 默认映射
        return new PlatformInputMapping
        {
            platform = platform,
            jumpButton = "Jump",
            attackButton = "Fire1",
            pauseKey = KeyCode.Escape
        };
    }

    void Update()
    {
        // 使用当前平台的输入映射
        bool jumpPressed = Input.GetButtonDown(currentMapping.jumpButton);
        bool attackPressed = Input.GetButtonDown(currentMapping.attackButton);
        bool pausePressed = Input.GetKeyDown(currentMapping.pauseKey);

        float horizontal = Input.GetAxis(currentMapping.horizontalAxis);
        float vertical = Input.GetAxis(currentMapping.verticalAxis);

        // 处理输入...
        if (jumpPressed) PerformJump();
        if (attackPressed) PerformAttack();
        if (pausePressed) TogglePauseMenu();

        MoveCharacter(horizontal, vertical);
    }

    // 移动端触摸输入处理
#if UNITY_IOS || UNITY_ANDROID
    void ProcessTouchInput()
    {
        if (Input.touchCount > 0)
        {
            Touch touch = Input.GetTouch(0);

            switch (touch.phase)
            {
                case TouchPhase.Began:
                    // 触摸开始
                    OnTouchStart(touch.position);
                    break;

                case TouchPhase.Moved:
                    // 触摸移动
                    OnTouchMove(touch.position, touch.deltaPosition);
                    break;

                case TouchPhase.Ended:
                case TouchPhase.Canceled:
                    // 触摸结束
                    OnTouchEnd(touch.position);
                    break;
            }
        }
    }

    // 移动端特定手势识别
    void ProcessGestures()
    {
        // 捏合手势检测
        if (Input.touchCount == 2)
        {
            Touch touch1 = Input.GetTouch(0);
            Touch touch2 = Input.GetTouch(1);

            Vector2 touch1PrevPos = touch1.position - touch1.deltaPosition;
            Vector2 touch2PrevPos = touch2.position - touch2.deltaPosition;

            float prevTouchDeltaMag = (touch1PrevPos - touch2PrevPos).magnitude;
            float touchDeltaMag = (touch1.position - touch2.position).magnitude;

            float deltaMagnitudeDiff = prevTouchDeltaMag - touchDeltaMag;

            // 检测捏合缩放
            if (Mathf.Abs(deltaMagnitudeDiff) > 10f)
            {
                if (deltaMagnitudeDiff > 0)
                    OnPinchZoom(-1); // 缩小
                else
                    OnPinchZoom(1);  // 放大
            }
        }
    }
#endif
}
```

### 2.2 自动化跨平台测试

自动化测试是跨平台兼容性保障的关键，特别是对于需要频繁测试多平台的项目。

**跨平台自动化测试架构**：
1. **测试代码平台无关性**：编写不依赖特定平台的测试逻辑
2. **平台适配层**：为每个平台提供特定的测试驱动和工具
3. **集中化测试管理**：统一调度和管理多平台测试执行
4. **结果聚合与分析**：跨平台测试结果统一分析和报告

**Unity跨平台自动化测试配置**：

```csharp
using NUnit.Framework;

using UnityEngine.TestTools;

public abstract class CrossPlatformTestBase
{

    protected abstract RuntimePlatform GetTargetPlatform();

    public virtual void Setup()
    {
        // 验证测试运行在正确的平台上
        if (Application.platform != GetTargetPlatform())

            Assert.Ignore($"此测试仅适用于 {GetPlatformName()} 平台");

        Debug.Log($"在 {GetPlatformName()} 平台上运行测试");

    {
        // 基础输入测试，各平台可重写
        Assert.IsTrue(InputSystemAvailable(), "输入系统应可用");
    }

    [Test]
    public virtual void Test_GraphicsAPI()
    {
        // 图形API测试
        var graphicsDevice = SystemInfo.graphicsDeviceType;
        Debug.Log($"图形设备: {graphicsDevice}");

        Assert.IsTrue(graphicsDevice != UnityEngine.Rendering.GraphicsDeviceType.Null,
            "图形设备应可用");
    }

    protected virtual bool InputSystemAvailable()
    {
        return Input.mousePresent || Input.touchSupported || Input.GetJoystickNames().Length > 0;
    }
}
```

```python
# AWS Device Farm自动化测试脚本
import boto3
import time
import json
from datetime import datetime

class AWSDeviceFarmTester:
    def __init__(self, project_arn, device_pool_arn):
        self.client = boto3.client('devicefarm', region_name='us-west-2')
        self.project_arn = project_arn
        self.device_pool_arn = device_pool_arn

    def upload_app(self, app_path):
        """上传应用APK或IPA文件"""
        with open(app_path, 'rb') as f:
            app_data = f.read()

        response = self.client.upload_project_artifact(
            projectArn=self.project_arn,
            name=f'app_{datetime.now().strftime("%Y%m%d_%H%M%S")}',
            type='ANDROID_APP',  # 或 IOS_APP
            contentType='application/octet-stream'
        )

        # 实际上传
        upload_arn = response['upload']['arn']
        # 这里需要实现分块上传逻辑

        return upload_arn

    def upload_tests(self, test_package_path):
        """上传测试包"""
        with open(test_package_path, 'rb') as f:
            test_data = f.read()

        response = self.client.upload_project_artifact(
            projectArn=self.project_arn,
            name=f'tests_{datetime.now().strftime("%Y%m%d_%H%M%S")}',
            type='INSTRUMENTATION_TEST_PACKAGE',
            contentType='application/octet-stream'
        )

        return response['upload']['arn']

    def schedule_run(self, app_arn, test_arn, device_configs):
        """安排测试运行"""
        response = self.client.schedule_run(
            projectArn=self.project_arn,
            appArn=app_arn,
            devicePoolArn=self.device_pool_arn,
            test={
                'type': 'INSTRUMENTATION',
                'testPackageArn': test_arn
            },
            configuration={
                'extraDataPackageArn': None,
                'networkProfileArn': None,
                'locale': 'en_US',
                'location': {
                    'latitude': 47.6205,
                    'longitude': -122.3493
                },
                'radios': {
                    'wifi': True,
                    'bluetooth': False,
                    'nfc': False,
                    'gps': True
                },
                'auxiliaryApps': [],
                'billingMethod': 'METERED',
                'customerArtifactPaths': None
            },
            name=f'TestRun_{datetime.now().strftime("%Y%m%d_%H%M%S")}',
            executionConfiguration={
                'jobTimeoutMinutes': 60,
                'accountsCleanup': True,
                'appPackagesCleanup': True,
                'videoCapture': True,
                'skipAppResign': False
            }
        )

        return response['run']['arn']

    def wait_for_completion(self, run_arn, poll_interval=30):
        """等待测试完成"""
        while True:
            response = self.client.get_run(arn=run_arn)
            status = response['run']['status']

            print(f"测试状态: {status}")

            if status in ['COMPLETED', 'ERRORED', 'STOPPED']:
                break

            time.sleep(poll_interval)

        return response['run']

    def get_results(self, run_arn):
        """获取测试结果"""
        response = self.client.get_run(arn=run_arn)

        # 获取测试结果详情
        jobs_response = self.client.list_jobs(arn=run_arn)

        results = {
            'status': response['run']['status'],
            'result': response['run']['result'],
            'counters': response['run']['counters'],
            'device_minutes': response['run']['deviceMinutes'],
            'jobs': []
        }

        for job in jobs_response['jobs']:
            job_details = self.client.get_job(arn=job['arn'])
            results['jobs'].append({
                'name': job['name'],
                'status': job['status'],
                'device': job_details['job']['device'],
                'counters': job_details['job']['counters']
            })

        return results

# 使用示例
```

## 3、平台特定功能测试

### 3.1 手柄输入测试

手柄是现代游戏的关键输入方式，需要跨平台一致的支持。

**手柄测试要点**：
1. **按钮映射测试**：验证所有按钮在游戏中正确响应
2. **轴输入测试**：摇杆、触发器输入的精度和死区处理
3. **振动反馈测试**：不同事件的振动效果和强度
4. **多手柄支持**：本地多人游戏的多手柄同时输入

**Unity手柄测试脚本**：
```csharp
using UnityEngine;
using UnityEngine.InputSystem;
using UnityEngine.InputSystem.Controls;
using UnityEngine.InputSystem.LowLevel;

public class GamepadTester : MonoBehaviour
{
    [System.Serializable]
    public class GamepadTestResult
    {
        public string gamepadName;
        public bool connected;
        public Dictionary<string, bool> buttonTests = new Dictionary<string, bool>();
        public Dictionary<string, float> axisTests = new Dictionary<string, float>();
        public Dictionary<string, bool> featureTests = new Dictionary<string, bool>();
        public List<string> issues = new List<string>();
    }

    public GamepadTestResult currentTestResult;
    private Gamepad currentGamepad;

    void Start()
    {
        // 监听手柄连接事件
        InputSystem.onDeviceChange += OnDeviceChange;

        // 开始测试第一个连接的手柄
        if (Gamepad.all.Count > 0)
        {
            StartTesting(Gamepad.all[0]);
        }
    }

    void OnDeviceChange(InputDevice device, InputDeviceChange change)
    {
        switch (change)
        {
            case InputDeviceChange.Added:
                Debug.Log($"设备连接: {device.name}");
                if (device is Gamepad)
                {
                    StartTesting(device as Gamepad);
                }
                break;

            case InputDeviceChange.Removed:
                Debug.Log($"设备断开: {device.name}");
                if (device == currentGamepad)
                {
                    currentTestResult.connected = false;
                    currentTestResult.issues.Add("设备在测试过程中断开");
                }
                break;
        }
    }

    void StartTesting(Gamepad gamepad)
    {
        currentGamepad = gamepad;
        currentTestResult = new GamepadTestResult
        {
            gamepadName = gamepad.name,
            connected = true
        };

        Debug.Log($"开始测试手柄: {gamepad.name}");

        // 开始各项测试
        StartCoroutine(RunButtonTests());
        StartCoroutine(RunAxisTests());
        StartCoroutine(RunFeatureTests());
    }

    System.Collections.IEnumerator RunButtonTests()
    {
        string[] buttonNames = { "A", "B", "X", "Y", "Start", "Select", "LeftShoulder", "RightShoulder" };

        foreach (var buttonName in buttonNames)
        {
            yield return StartCoroutine(TestButton(buttonName));
        }
    }

    System.Collections.IEnumerator TestButton(string buttonName)
    {
        ButtonControl button = null;

        switch (buttonName)
        {
            case "A": button = currentGamepad.aButton; break;
            case "B": button = currentGamepad.bButton; break;
            case "X": button = currentGamepad.xButton; break;
            case "Y": button = currentGamepad.yButton; break;
            case "Start": button = currentGamepad.startButton; break;
            case "Select": button = currentGamepad.selectButton; break;
            case "LeftShoulder": button = currentGamepad.leftShoulder; break;
            case "RightShoulder": button = currentGamepad.rightShoulder; break;
        }

        if (button == null)
        {
            currentTestResult.buttonTests[buttonName] = false;
            currentTestResult.issues.Add($"按钮 {buttonName} 不可用");
            yield break;
        }

        Debug.Log($"按下 {buttonName} 按钮...");

        bool pressed = false;
        float startTime = Time.time;

        // 等待按钮被按下
        while (Time.time - startTime < 5f)
        {
            if (button.isPressed)
            {
                pressed = true;
                break;
            }
            yield return null;
        }

        currentTestResult.buttonTests[buttonName] = pressed;

        if (!pressed)
            currentTestResult.issues.Add($"按钮 {buttonName} 未响应");
        else
            Debug.Log($"按钮 {buttonName} 测试通过");
    }

    System.Collections.IEnumerator RunAxisTests()
    {
        yield return StartCoroutine(TestAxis("LeftStick", currentGamepad.leftStick));
        yield return StartCoroutine(TestAxis("RightStick", currentGamepad.rightStick));
        yield return StartCoroutine(TestTrigger("LeftTrigger", currentGamepad.leftTrigger));
        yield return StartCoroutine(TestTrigger("RightTrigger", currentGamepad.rightTrigger));
    }

    System.Collections.IEnumerator TestAxis(string axisName, StickControl stick)
    {
        Debug.Log($"测试摇杆 {axisName}...");

        // 测试各个方向
        Vector2[] testDirections = {
            Vector2.up, Vector2.down, Vector2.left, Vector2.right,
            new Vector2(0.7f, 0.7f), new Vector2(-0.7f, 0.7f)
        };

        foreach (var direction in testDirections)
        {
            Debug.Log($"  请将{axisName}摇杆推向 {direction} 方向");

            float startTime = Time.time;
            bool detected = false;

            while (Time.time - startTime < 5f)
            {
                Vector2 current = stick.ReadValue();

                // 检查是否接近测试方向
                if (Vector2.Dot(current.normalized, direction.normalized) > 0.8f &&
                    current.magnitude > 0.5f)
                {
                    detected = true;
                    currentTestResult.axisTests[$"{axisName}_{direction}"] = current.magnitude;
                    break;
                }

                yield return null;
            }

            if (!detected)
            {
                currentTestResult.issues.Add($"摇杆 {axisName} {direction} 方向响应异常");
            }
        }
    }

    System.Collections.IEnumerator RunFeatureTests()
    {
        // 测试振动功能
        if (currentGamepad.vibration != null)
        {
            yield return StartCoroutine(TestVibration());
        }

        // 测试触摸板（如DualShock/DualSense）
        if (currentGamepad is DualShockGamepad || currentGamepad is DualSenseGamepad)
        {
            yield return StartCoroutine(TestTouchpad());
        }
    }

    System.Collections.IEnumerator TestVibration()
    {
        Debug.Log("测试振动功能...");

        // 测试不同强度的振动
        float[] intensities = { 0.3f, 0.6f, 1.0f };

        foreach (float intensity in intensities)
        {
            currentGamepad.SetMotorSpeeds(intensity, intensity);
            Debug.Log($"  振动强度: {intensity}");
            yield return new WaitForSeconds(1f);
        }

        // 停止振动
        currentGamepad.SetMotorSpeeds(0f, 0f);
        currentTestResult.featureTests["Vibration"] = true;
    }

    void OnDestroy()
    {
        InputSystem.onDeviceChange -= OnDeviceChange;
    }
}
```

### 4.2 合规性自动化测试

自动化合规性检查可以提前发现潜在问题，减少认证失败风险。

**合规性测试自动化工具**：
```python
# 简单的合规性检查脚本
import os
import json
import re
from pathlib import Path

class ComplianceChecker:
    def __init__(self, game_root_path):
        self.game_root = Path(game_root_path)
        self.issues = []

    def check_all(self):
        """运行所有合规性检查"""
        print("开始合规性检查...")

        self.check_file_structure()
        self.check_naming_conventions()
        self.check_content_guidelines()
        self.check_performance_baseline()
        self.check_localization()

        self.report_results()

    def check_file_structure(self):
        """检查文件结构合规性"""
        required_dirs = ["Assets", "Scripts", "Scenes", "Resources"]
        for dir_name in required_dirs:
            dir_path = self.game_root / dir_name
            if not dir_path.exists():
                self.issues.append(f"缺失必需目录: {dir_name}")

    def check_naming_conventions(self):
        """检查命名规范"""
        # 检查资源文件命名
        for file_path in self.game_root.rglob("*.asset"):
            filename = file_path.name
            if not re.match(r'^[a-z][a-z0-9_]*\.asset$', filename.lower()):
                self.issues.append(f"资源文件命名不规范: {filename}")

        # 检查场景文件命名
        for file_path in self.game_root.rglob("*.unity"):
            if not re.match(r'^[A-Z][a-zA-Z0-9_]*\.unity$', file_path.name):
                self.issues.append(f"场景文件命名不规范: {file_path.name}")

    def check_content_guidelines(self):
        """检查内容指南合规性"""
        # 检查不当内容
        inappropriate_keywords = ["暴力", "色情", "仇恨言论"]  # 示例关键词
        text_files = list(self.game_root.rglob("*.txt")) + \
                     list(self.game_root.rglob("*.json")) + \
                     list(self.game_root.rglob("*.xml"))

        for file_path in text_files:
            try:
                content = file_path.read_text(encoding='utf-8', errors='ignore')
                for keyword in inappropriate_keywords:
                    if keyword in content:
                        self.issues.append(f"发现不当内容 '{keyword}' 在文件: {file_path}")
            except:
                pass  # 跳过无法读取的文件

    def check_performance_baseline(self):
        """检查性能基准"""
        # 检查性能配置文件
        perf_config = self.game_root / "Config" / "performance.json"
        if perf_config.exists():
            try:
                with open(perf_config, 'r', encoding='utf-8') as f:
                    config = json.load(f)

                # 验证性能目标
                if "target_fps" not in config or config["target_fps"] < 30:
                    self.issues.append("性能目标未设置或低于30fps")

                if "memory_budget" not in config:
                    self.issues.append("未设置内存预算")

            except json.JSONDecodeError:
                self.issues.append("性能配置文件格式错误")

    def check_localization(self):
        """检查本地化支持"""
        loc_dir = self.game_root / "Localization"
        if not loc_dir.exists():
            self.issues.append("未找到本地化目录")
            return

        # 检查必需的语言文件
        required_langs = ["en", "zh", "ja"]  # 根据目标市场调整
        for lang in required_langs:
            lang_file = loc_dir / f"{lang}.json"
            if not lang_file.exists():
                self.issues.append(f"缺失本地化文件: {lang}")

    def report_results(self):
        """报告检查结果"""
        if not self.issues:
            print("✓ 所有合规性检查通过")
            return

        print(f"发现 {len(self.issues)} 个合规性问题:")
        for i, issue in enumerate(self.issues, 1):
            print(f"  {i}. {issue}")

# 使用示例
if __name__ == "__main__":
    checker = ComplianceChecker("/path/to/your/game")
    checker.check_all()
```

## 总结

跨平台测试与兼容性是现代游戏开发中不可或缺的环节。通过系统化的测试策略、自动化工具和深入的平台特性理解，独立开发者可以在有限资源下确保游戏在所有目标平台上提供优质体验。记住跨平台测试的核心原则：**早期测试、频繁测试、自动化重复测试、优先处理影响最大的兼容性问题**。建立适合自己项目规模和目标平台的测试体系，从开发初期就考虑跨平台兼容性，最终交付让所有平台玩家都能享受的高质量游戏。
