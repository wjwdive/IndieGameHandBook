## 1、3D美术基础

### 1.1 3D美术核心概念

**三维空间的基本元素**：
1. **顶点（Vertex）**：3D空间中的点，定义形状的基本单位
2. **边（Edge）**：连接两个顶点的线
3. **面（Face/Polygon）**：由三个或更多顶点定义的平面区域
4. **网格（Mesh）**：顶点、边和面的集合，构成3D模型
5. **UV坐标**：将3D表面映射到2D纹理的坐标系统

**3D美术与传统3D的区别**：
游戏3D美术不仅是艺术创作，更是技术实现。与传统3D艺术相比，游戏3D美术需要：

- **实时渲染约束**：必须在1/60秒内完成渲染
- **性能预算**：面数、纹理大小、骨骼数量有限制
- **交互性需求**：模型需要支持动画、碰撞、变形
- **平台适配**：在不同硬件上保持视觉质量和性能平衡

## 2、3D建模技术

### 2.1 多边形建模基础

**多边形建模的核心概念**：

**面数（Polygon Count）管理**：
```python
class PolygonBudget:
    """多边形预算管理"""

    def __init__(self, total_budget=100000):
        self.total_budget = total_budget  # 总面数预算
        self.used_polygons = 0
        self.models = {}

    def allocate_polygons(self, model_name, importance_level):
        """根据重要性分配面数"""
        allocation_ratios = {
            "hero": 0.20,      # 主角：20%预算
            "major_npc": 0.10, # 主要NPC：10%
            "minor_npc": 0.05, # 次要NPC：5%
            "environment": 0.40, # 环境：40%
            "props": 0.25      # 道具：25%
        }

        allocated = int(self.total_budget * allocation_ratios.get(importance_level, 0.05))

        if self.used_polygons + allocated <= self.total_budget:
            self.models[model_name] = {
                "budget": allocated,
                "importance": importance_level,
                "actual_polygons": 0
            }
            self.used_polygons += allocated
            return allocated
        else:
            # 重新分配预算
            return self.reallocate_budget(model_name, importance_level)
```

**建模基本原则**：
1. **四边形优先**：尽量使用四边形而非三角形
2. **均匀分布**：网格密度应与细节需求匹配
3. **流线型拓扑**：边流向应支持预期变形
4. **简化几何**：移除玩家看不见的内部面和隐藏面

**低多边形建模技巧**：

```python
import os

def bake_normal_map(high_poly_obj, low_poly_obj, output_path):
    """烘焙法线贴图"""

    # 选择低模作为活动对象
    bpy.context.view_layer.objects.active = low_poly_obj
    low_poly_obj.select_set(True)

    # 设置烘焙选项
    bpy.context.scene.render.engine = 'CYCLES'
    bpy.context.scene.cycles.samples = 32  # 采样数

    # 设置图像纹理
    image_name = f"{low_poly_obj.name}_normal"
    image_size = 2048  # 贴图大小

    if image_name not in bpy.data.images:
        bpy.ops.image.new(
            name=image_name,
            width=image_size,
            height=image_size,
            color=(0.5, 0.5, 1.0, 1.0),  # 法线贴图基础色
            alpha=True
        )

    # 配置材质和节点
    setup_bake_material(low_poly_obj, image_name)

    # 执行烘焙
    bpy.ops.object.bake(
        type='NORMAL',
        save_mode='EXTERNAL',
        filepath=output_path,
        width=image_size,
        height=image_size,
        margin=16,  # 边距
        use_selected_to_active=True,
        cage_extrusion=0.1,  # 笼子挤出
        max_ray_distance=0.1
    )

    print(f"法线贴图烘焙完成：{output_path}")
```

## 3、纹理与材质

### 3.1 UV展开与纹理映射

**UV展开基础**：

**UV坐标系统**：
- **U轴**：水平方向（0到1）
- **V轴**：垂直方向（0到1）
- **UV岛（UV Island）**：模型中连续展开的部分
- **接缝（Seam）**：展开时切割的边缘

**良好UV展开的原则**：
1. **最小化拉伸**：UV面应与3D面比例匹配
2. **最大化利用率**：尽可能填满UV空间
3. **接缝隐藏**：将接缝放在不易见的位置
4. **纹理分辨率一致**：所有部分使用相近的纹理密度

**独立游戏UV策略**：

**自动展开与手动调整结合**：
```python
def optimize_uv_layout(model):
    """优化UV布局的实用函数"""

    optimizations = []

    # 1. 智能展开
    if model.complexity > 0.7:
        # 复杂模型：按材质和区域分离
        uv_islands = separate_by_material(model)
        optimizations.append("按材质分离UV岛")
    else:
        # 简单模型：自动展开
        uv_islands = auto_unwrap(model)
        optimizations.append("自动UV展开")

    # 2. 调整缩放确保纹理密度一致
    for island in uv_islands:
        target_density = calculate_target_density(island.importance)
        scale_to_density(island, target_density)

    optimizations.append("统一纹理密度")

    # 3. 自动排列UV岛
    packed_islands = pack_uv_islands(uv_islands, padding=0.02)
    optimizations.append(f"UV排列效率: {packed_islands.efficiency:.1%}")

    # 4. 接缝优化
    visible_seams = check_seam_visibility(model, packed_islands)
    if visible_seams:
        optimizations.append(f"需要手动调整{len(visible_seams)}个可见接缝")

    return {
        "uv_islands": packed_islands,
        "optimizations": optimizations,
        "texture_utilization": packed_islands.efficiency
    }
```

**纹理图集（Texture Atlas）策略**：

**优点**：
- 减少绘制调用
- 优化内存使用
- 简化材质管理

**缺点**：
- UV展开更复杂
- 局部更新困难
- 可能造成纹理浪费

**独立游戏图集建议**：
1. **按材质类型分组**：金属、木材、布料等
2. **动态与静态分离**：角色用动态图集，环境用静态图集
3. **分辨率分级**：重要元素高分辨率，次要元素低分辨率
4. **预留空间**：为未来更新预留空白区域

```gdscript
// 纹理输入
uniform sampler2D albedo_texture : source_color;
uniform sampler2D normal_texture : hint_normal;
uniform sampler2D roughness_texture : hint_default_white;
uniform sampler2D metallic_texture : hint_default_black;

uniform float metallic_multiplier : hint_range(0, 1) = 1.0;

void fragment() {

    vec3 normal_map = texture(normal_texture, UV).rgb;
    float roughness = texture(roughness_texture, UV).r * roughness_multiplier;
    float metallic = texture(metallic_texture, UV).r * metallic_multiplier;

    // 设置材质属性
    ALBEDO = albedo_color.rgb;
    NORMAL_MAP = normal_map;
    ROUGHNESS = roughness;
    METALLIC = metallic;

    // 简单性能优化：根据距离调整效果
    float distance_factor = clamp(1.0 - CAMERA_POSITION.y / 100.0, 0.0, 1.0);
```

```glsl
vec3 calculateLowPolyLighting(vec3 normal, vec3 lightDir) {
    // 平面着色：每个面统一颜色
    float diffuse = max(0.0, dot(normal, lightDir));

    // 色阶化：将连续光照离散化
    float steps = 3.0; // 3级明暗
```

```gdscript
# Godot中程序化头部跟踪
extends Spatial

export(NodePath) var target_path
export(float) var rotation_speed = 5.0
export(float) var max_rotation = 45.0  # 最大旋转角度

var target: Spatial
var neck: Bone
var head: Bone

func _ready():
    target = get_node(target_path)
    neck = get_node("Armature/Skeleton/Neck")
    head = get_node("Armature/Skeleton/Head")

func _process(delta):
    if not target or not neck or not head:
        return

    # 计算看向目标的方向
    var head_global = head.global_transform.origin
    var target_direction = (target.global_transform.origin - head_global).normalized()

    # 转换为颈部局部空间
    var neck_basis = neck.global_transform.basis
    var local_direction = neck_basis.xform_inv(target_direction)

    # 计算旋转角度（限制在最大范围内）
    var target_rotation = Vector2(
        -atan2(local_direction.z, local_direction.x),  # 水平旋转
        atan2(local_direction.y, sqrt(local_direction.x * local_direction.x + local_direction.z * local_direction.z))  # 垂直旋转
    )

    target_rotation.x = clamp(target_rotation.x, deg2rad(-max_rotation), deg2rad(max_rotation))
    target_rotation.y = clamp(target_rotation.y, deg2rad(-max_rotation/2), deg2rad(max_rotation/2))

    # 平滑旋转
    var current_rotation = Vector2(neck.rotation.y, head.rotation.x)
    var new_rotation = current_rotation.linear_interpolate(target_rotation, rotation_speed * delta)

    neck.rotation.y = new_rotation.x
    head.rotation.x = new_rotation.y
```

2. **优化思维**：始终考虑性能影响，在艺术效果和技术限制间找到平衡。
3. **迭代开发**：通过快速原型和测试验证美术方案的有效性。
4. **工具精通**：深入掌握1-2个核心工具，而非浅尝辄止多个工具。
5. **资源重用**：建立可重用的素材库、材质库和动画库。

**给独立开发者的实用建议**：

1. **从低多边形开始**：低多边形风格技术门槛较低，适合学习和原型制作。
2. **关注核心资产**：优先制作玩家最常看到的角色和环境。
3. **学习基础原理**：理解建模、纹理、动画的基础原理比掌握特定工具更重要。
4. **参与社区**：3D艺术社区资源丰富，学习他人经验可以少走弯路。
5. **保持耐心**：3D美术制作是长期过程，设定现实的时间预期。

**技术发展趋势**：
1. **实时渲染进步**：引擎实时渲染能力不断提升，降低了对预烘焙的依赖。
2. **程序化内容**：Houdini、Substance等工具使程序化内容创作更加普及。
3. **AI辅助工具**：AI在概念生成、纹理创建、动画辅助等方面发挥作用。
4. **云协作**：基于云的3D工具和工作流使远程协作更加容易。

记住：在独立游戏开发中，**一致性比真实性更重要，风格比细节更重要，完成度比完美度更重要**。一个风格统一、运行流畅的简单3D游戏，远比一个技术华丽但永远无法完成的复杂项目更有价值。

## 动手实践

1. **低多边形角色制作**：
   - 使用Blender创建一个500-1000面的风格化角色
   - 制作完整的UV展开和简单纹理
   - 创建基础骨骼绑定和待机动画
   - 导出到游戏引擎并实现基础控制

2. **模块化环境套件**：
   - 设计一套模块化的低多边形建筑部件
   - 确保所有部件使用统一的网格密度和纹理尺寸
   - 在引擎中使用部件构建复杂场景
   - 实现LOD系统和遮挡剔除

3. **PBR材质创建**：
   - 使用Substance Painter或类似工具创建PBR材质
   - 制作至少5种不同表面类型（金属、木材、石材等）
   - 在游戏引擎中测试材质在不同光照下的表现
   - 优化材质以适应移动平台

4. **动画状态机实现**：
   - 为角色创建完整的动画状态机
   - 包括待机、行走、奔跑、跳跃、攻击等状态
   - 实现平滑的状态过渡和动画混合
   - 添加程序化动画增强效果
