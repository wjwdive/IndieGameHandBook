# 数值设计基础

## 概述

本章将介绍游戏数值设计的基本原则、方法和工具。数值设计是游戏平衡的核心，它决定了游戏的节奏、难度曲线、经济系统和长期可玩性。优秀的数值设计让游戏感觉"正确"——挑战与奖励平衡，成长与目标匹配，选择有意义。

**学习目标**：
- 理解数值设计在游戏体验中的关键作用
- 掌握游戏数值平衡的基本原理和数学模型
- 学习角色属性、伤害公式、经济系统的设计方法
- 掌握使用电子表格和工具进行数值建模的技术
- 能够为不同类型游戏设计合适的数值系统
- 了解数值测试、迭代和平衡调整的方法

## 1、数值设计基础概念

### 1.1 什么是数值设计

**数值设计（Numerical Design）** 是创建和管理游戏中所有数值系统的过程，包括：
- **角色属性**：生命值、攻击力、防御力等
- **成长系统**：经验值、等级、技能点
- **经济系统**：货币、物品价格、资源产出
- **战斗公式**：伤害计算、命中率、暴击
- **时间系统**：冷却时间、生产时间、游戏节奏

> **注意**：数值设计是游戏的"隐形架构"。玩家通常不会注意到好的数值设计，但一定会注意到差的数值设计——表现为游戏太简单、太难、太磨人或太随机。

### 1.2 数值设计的目标

**成功的数值设计应实现**：
1. **平衡性**：所有选项都有合理价值，无绝对最优解
2. **可预测性**：玩家能理解数值关系，做出明智决策
3. **渐进感**：成长和进步感觉有意义且满足
4. **节奏控制**：游戏进程有适当的速度和里程碑
5. **情感曲线**：数值变化支持期望的情感体验（紧张、放松、成就感）

**数值设计评价维度**：
- **内部一致性**：数值系统内部逻辑自洽
- **外部合理性**：数值与游戏世界观和主题匹配
- **玩家感知**：数值变化对玩家体验的实际影响
- **长期健康**：系统支持长期游戏而非短期利用

## 2、数值设计核心原理

### 2.1 边际效用与平衡

**边际效用原理**：随着某属性增加，每单位增加带来的收益递减。

**示例：防御力设计**：
```python
def calculate_damage_reduction(defense, damage):
    """计算防御力带来的伤害减免"""
    # 线性模型：简单但可能导致后期无效
    # reduction = defense * 0.01  # 每点防御减少1%伤害

    # 曲线模型：边际效用递减
    reduction = 1 - (1 / (1 + defense * 0.02))  # 防御越高，每点收益越小

    actual_damage = damage * (1 - reduction)
    return actual_damage

# 测试不同防御值的效果
for defense in [0, 10, 50, 100, 200]:
    damage_received = calculate_damage_reduction(defense, 100)
    print(f"防御{defense}: 承受伤害{damage_received:.1f}")
```

**边际效用设计原则**：
1. **避免线性无限增长**：线性增长导致后期数值爆炸
2. **软上限设计**：属性越高提升越困难
3. **机会成本平衡**：投资一个属性意味着放弃其他属性
4. **情境价值**：同一属性在不同情境下有不同价值

### 2.2 数值缩放与曲线

游戏中的数值通常随时间/等级增长，需要合理缩放。

**常见缩放曲线**：
- **线性增长**：每次增加固定值（简单但可能乏味）
- **指数增长**：每次按比例增加（适合长期游戏）
- **对数增长**：早期增长快，后期慢（适合平滑体验）
- **S型曲线**：开始慢，中间快，最后慢（自然感觉）

**经验值需求曲线示例**：
```python
def experience_for_level(level, curve_type="exponential"):
    """计算升到指定等级所需经验值"""
    base_xp = 1000

    if curve_type == "linear":
        return base_xp * level
    elif curve_type == "exponential":
        growth = 1.5  # 指数增长系数
        return int(base_xp * (growth ** (level - 1)))
    elif curve_type == "logarithmic":
        # 对数增长：早期升级快，后期慢
        return int(base_xp * (1 + math.log(level) * 2))
    elif curve_type == "s_curve":
        # S型曲线：平滑过渡
        return int(base_xp * (level ** 1.8) / (1 + 0.1 * level))

# 比较不同曲线的前10级需求
for level in range(1, 11):
    xp_exp = experience_for_level(level, "exponential")
    xp_log = experience_for_level(level, "logarithmic")
    print(f"等级{level}: 指数{xp_exp} | 对数{xp_log}")
```

**曲线选择指南**：
| 游戏类型 | 推荐曲线 | 理由 | 风险 |
|---------|---------|------|------|
| **休闲游戏** | 对数曲线 | 早期快速满足感，避免后期过难 | 可能缺乏长期目标 |
| **RPG/MMO** | 指数曲线 | 提供明确长期目标，支持内容消耗 | 新手可能感到沮丧 |
| **策略游戏** | S型曲线 | 平滑进展，避免中期乏味 | 需要精细调整 |
| **竞技游戏** | 线性/软上限 | 保持比赛公平性，避免雪球效应 | 可能缺乏成长感 |

### 2.3 随机性与确定性

游戏中需要平衡随机性（运气）和确定性（技能）。

**随机性类型**：
- **输入随机性**：游戏开始时的随机因素（地图生成、起始资源）
- **输出随机性**：玩家行动结果的随机因素（命中率、暴击、掉落）
- **元随机性**：影响游戏规则的随机因素（每日任务、限时事件）

**随机性设计原则**：
1. **技能主导**：玩家决策应比随机因素更重要
2. **可控风险**：玩家能评估和应对随机结果
3. **失败学习**：随机失败应提供信息而非纯粹惩罚
4. **长期公平**：随机性应在大量游戏中趋于公平

**命中率计算公式优化**：
```python
def calculate_hit_chance(attacker_accuracy, defender_evasion, base_chance=0.9):
    """计算命中率，避免极端情况"""
    # 简单减法可能导致0%或100%+
    # chance = base_chance + (attacker_accuracy - defender_evasion) * 0.01

    # 更好的公式：渐进逼近极限
    difference = attacker_accuracy - defender_evasion
    scaling_factor = 0.02  # 每点差异影响2%

    # 使用曲线函数，避免极端值
    chance = base_chance / (1 + math.exp(-difference * scaling_factor))

    # 设置软上下限
    chance = max(0.05, min(0.95, chance))  # 5%-95%之间

    return chance

# 测试不同情况
test_cases = [
    (50, 50, "势均力敌"),
    (80, 20, "明显优势"),
    (20, 80, "明显劣势")
]

for acc, eva, desc in test_cases:
    chance = calculate_hit_chance(acc, eva)
    print(f"{desc}: 命中率{chance:.1%}")
```

### 3.1 角色属性系统

**基础属性设计模板**：
```python
class CharacterAttributes:
    """角色基础属性系统"""
    def __init__(self, level=1):
        self.level = level
        self.base_stats = {
            "strength": 10,      # 力量：影响物理攻击
            "dexterity": 10,     # 敏捷：影响速度、暴击
            "constitution": 10,  # 体质：影响生命值
            "intelligence": 10,  # 智力：影响魔法攻击
            "wisdom": 10,        # 智慧：影响魔法防御
            "luck": 10           # 幸运：影响随机事件
        }

        self.derived_stats = self.calculate_derived_stats()

    def calculate_derived_stats(self):
        """根据基础属性计算衍生属性"""
        str = self.base_stats["strength"]
        dex = self.base_stats["dexterity"]
        con = self.base_stats["constitution"]
        intel = self.base_stats["intelligence"]
        wis = self.base_stats["wisdom"]
        luck = self.base_stats["luck"]

        return {
            "max_health": 50 + (con * 5) + (self.level * 10),
            "physical_attack": str * 2,
            "magical_attack": intel * 2,
            "physical_defense": con * 1.5 + str * 0.5,
            "magical_defense": wis * 1.5 + intel * 0.5,
            "critical_chance": 0.05 + (dex * 0.002) + (luck * 0.001),
            "critical_damage": 1.5 + (dex * 0.01),
            "evasion": 0.05 + (dex * 0.005),
            "attack_speed": 1.0 + (dex * 0.02)
        }

    def level_up(self):
        """升级增加属性"""
        self.level += 1
        # 每级增加属性点
        points_to_allocate = 5

        # 自动分配或玩家分配逻辑
        # 这里简化：平均增长
        for stat in self.base_stats:
            self.base_stats[stat] += 1

        self.derived_stats = self.calculate_derived_stats()
```

**属性平衡原则**：
1. **明确角色定位**：不同职业/角色类型有不同属性重点
2. **属性互补**：确保没有"全能"属性，所有属性都有价值
3. **软上限设计**：属性越高，每点收益越低
4. **情境价值**：不同游戏阶段不同属性重要性变化

### 3.2 战斗数值系统

**伤害计算公式设计**：
```python
class CombatCalculator:
    """战斗数值计算器"""

    @staticmethod
    def calculate_damage(attacker, defender, skill=None):
        """计算伤害"""
        # 基础攻击和防御
        attack = attacker.stats.get("physical_attack", 10)
        defense = defender.stats.get("physical_defense", 5)

        # 基础伤害公式
        base_damage = max(1, attack - defense * 0.5)

        # 随机浮动（±10%）
        damage_variation = random.uniform(0.9, 1.1)
        base_damage *= damage_variation

        # 技能倍率
        skill_multiplier = skill["multiplier"] if skill else 1.0
        base_damage *= skill_multiplier

        # 暴击检查
        crit_chance = attacker.stats.get("critical_chance", 0.05)
        if random.random() < crit_chance:
            crit_multiplier = attacker.stats.get("critical_damage", 1.5)
            base_damage *= crit_multiplier
            is_critical = True
        else:
            is_critical = False

        # 防御方特殊效果（格挡、闪避等）
        evasion = defender.stats.get("evasion", 0.05)
        if random.random() < evasion:
            return 0, "missed"  # 未命中

        # 最终伤害取整
        final_damage = int(base_damage)

        return final_damage, "critical" if is_critical else "normal"

    @staticmethod
    def simulate_combat(attacker, defender, rounds=10):
        """模拟多次战斗，用于平衡测试"""
        results = {
            "attacker_wins": 0,
            "defender_wins": 0,
            "average_rounds": 0,
            "damage_dealt": []
        }

        for _ in range(100):  # 模拟100场战斗
            attacker_hp = attacker.stats["max_health"]
            defender_hp = defender.stats["max_health"]
            rounds_taken = 0

            while attacker_hp > 0 and defender_hp > 0 and rounds_taken < 50:
                # 攻击方回合
                damage, _ = CombatCalculator.calculate_damage(attacker, defender)
                defender_hp -= damage
                rounds_taken += 1

                if defender_hp <= 0:
                    results["attacker_wins"] += 1
                    break

                # 防御方回合
                damage, _ = CombatCalculator.calculate_damage(defender, attacker)
                attacker_hp -= damage
                rounds_taken += 1

                if attacker_hp <= 0:
                    results["defender_wins"] += 1
                    break

            results["average_rounds"] += rounds_taken
            results["damage_dealt"].append(attacker_hp - defender_hp)

        # 计算平均值
        results["average_rounds"] /= 100
        results["win_rate"] = results["attacker_wins"] / 100

        return results
```

**战斗平衡检查清单**：
- [ ] 战斗是否在合理回合数内结束（通常3-10回合）？
- [ ] 先手优势是否过大？是否有平衡机制？
- [ ] 暴击/闪避等随机因素是否过于主导？
- [ ] 防御和生命值比例是否合理（避免刮痧或秒杀）？

### 3.3 经济系统设计

游戏经济系统管理资源的产出、消耗和流通。

**经济系统核心要素**：
```python
class EconomySimulator:
    """经济系统模拟器"""

    def __init__(self):
        self.currencies = {
            "gold": {"supply": 10000, "demand": 1.0},
            "gems": {"supply": 1000, "demand": 5.0},
            "honor": {"supply": 5000, "demand": 2.0}
        }

        self.items = {
            "health_potion": {"base_price": 50, "stock": 100},
            "mana_potion": {"base_price": 75, "stock": 80},
            "basic_sword": {"base_price": 200, "stock": 50},
            "rare_sword": {"base_price": 1000, "stock": 10}
        }

    def calculate_dynamic_price(self, item_name, player_wealth=None):
        """计算动态价格（基于供需）"""
        item = self.items[item_name]
        base_price = item["base_price"]
        stock = item["stock"]

        # 供需调整
        demand_factor = 1.0
        if player_wealth:
            # 玩家越富有，价格可能越高（假设需求增加）
            demand_factor = min(2.0, 1.0 + player_wealth / 10000)

        # 库存调整
        stock_factor = max(0.5, min(2.0, 100 / max(1, stock)))

        # 最终价格
        final_price = base_price * demand_factor * stock_factor

        return int(final_price)

    def simulate_economy_cycle(self, players=100, days=30):
        """模拟经济周期"""
        economy_data = []

        for day in range(days):
            daily_transactions = 0
            total_gold_circulation = 0

            # 模拟玩家活动
            for player in range(players):
                # 玩家获取资源（简化模型）
                gold_earned = random.randint(50, 200)
                self.currencies["gold"]["supply"] += gold_earned

                # 玩家消费（概率性）
                if random.random() < 0.3:  # 30%玩家购物
                    item = random.choice(list(self.items.keys()))
                    price = self.calculate_dynamic_price(item)

                    # 简化：假设玩家有足够金钱
                    self.currencies["gold"]["supply"] -= price
                    self.items[item]["stock"] -= 1

                    daily_transactions += 1
                    total_gold_circulation += price

            # 记录每日数据
            economy_data.append({
                "day": day,
                "total_supply": self.currencies["gold"]["supply"],
                "transactions": daily_transactions,
                "gold_circulation": total_gold_circulation,
                "average_price_health": self.calculate_dynamic_price("health_potion")
            })

        return economy_data
```

**经济系统设计原则**：
1. **资源循环**：创造有意义的资源消耗和再生产循环
2. **通货膨胀控制**：避免货币无限累积导致贬值
3. **价值锚点**：提供明确的价值参考标准
4. **玩家驱动**：经济应由玩家活动驱动，而非纯粹脚本

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

class BalanceTester:
    """数值平衡测试工具"""

    def __init__(self):
        self.test_results = []

    def test_attribute_balance(self, attributes_range, simulations=1000):
        """测试属性平衡性"""
        results = []

        for str_val in attributes_range:
            for dex_val in attributes_range:
                win_rate = self.simulate_attribute_matchup(
                    strength=str_val, dexterity=dex_val,
                    simulations=simulations

                    "strength": str_val,
                    "dexterity": dex_val,
                    "win_rate": win_rate

            char2_power = char2["str"] * 1.0 + char2["dex"] * 0.5

            if char1_power > char2_power:
                wins += 1

        return wins / simulations

### 4.3 迭代与调整方法

**数值迭代流程**：
```
1. 建立基线模型 → 2. 设置测试标准 → 3. 运行模拟测试 → 4. 分析结果 → 5. 调整参数
      ↓                 ↓                ↓              ↓           ↓
定义核心公式     确定成功指标     自动化测试    识别不平衡点   修改系数/公式
设定初始值       量化目标         收集数据     分析原因       验证效果
```

**常见数值问题与调整**：

| 问题症状 | 可能原因 | 调整策略 | 验证方法 |
|---------|---------|---------|---------|
| **后期数值爆炸** | 指数增长太快 | 降低增长率，增加软上限 | 模拟100级数值 |
| **属性单一最优** | 某属性收益过高 | 增加边际递减，强化其他属性 | 计算属性价值比 |
| **战斗时间过长** | 防御/生命值过高 | 调整伤害公式，增加穿透机制 | 统计平均回合数 |
| **经济通货膨胀** | 货币产出过多 | 增加货币消耗，调整掉落率 | 跟踪物价变化 |

**A/B测试框架**：
```python
def ab_test_numeric_changes(player_data, version_a, version_b):
    """A/B测试数值调整效果"""

    # 随机分配玩家到不同版本
    group_a = []
    group_b = []

    for player in player_data:
        if random.random() < 0.5:
            group_a.append(apply_version(player, version_a))
        else:
            group_b.append(apply_version(player, version_b))

    # 收集指标
    metrics_a = collect_metrics(group_a)
    metrics_b = collect_metrics(group_b)

    # 统计分析
    statistical_significance = calculate_significance(metrics_a, metrics_b)

    return {
        "version_a": metrics_a,
        "version_b": metrics_b,
        "difference": calculate_difference(metrics_a, metrics_b),
        "significance": statistical_significance,
        "recommendation": "A" if metrics_a["overall_score"] > metrics_b["overall_score"] else "B"
    }
```

## 5、不同类型游戏的数值设计

### 5.1 RPG数值设计要点

**RPG数值设计层次**：
1. **角色成长曲线**：经验值、等级、属性点分配
2. **装备数值体系**：武器伤害、防具防御、稀有度加成
3. **技能数值平衡**：伤害公式、冷却时间、资源消耗
4. **敌人数值配置**：生命值、攻击力、特殊能力
5. **经济系统**：货币获取、物品价格、交易平衡

**RPG数值平衡检查表**：
- [ ] 玩家等级与敌人等级匹配度（±3级内）
- [ ] 装备升级提升感（每级提升10-30%）
- [ ] 技能组合多样性（无绝对最优build）
- [ ] 资源管理有意义（魔力/体力限制策略选择）
- [ ] 成长曲线平滑（无明显卡点或突进）

### 5.2 策略游戏数值设计

**策略游戏数值特点**：
- **对称 vs 不对称平衡**：相同起点或不同但平衡的派系
- **科技树设计**：研究成本与收益的平衡
- **单位克制系统**：石头剪刀布式的克制关系
- **地图资源分布**：资源点位置影响开局策略

**单位数值设计模板**：
```python
strategy_unit_template = {
    "name": "步兵",
    "cost": {"gold": 100, "food": 1, "time": 30},
    "combat_stats": {
        "health": 100,
        "attack": 15,
        "defense": 5,
        "range": 1,
        "speed": 3,
        "vision": 4
    },
    "bonus_damage": {
        "cavalry": 1.5,  # 对骑兵造成150%伤害
        "archers": 0.75  # 对弓箭手造成75%伤害
    },
    "abilities": [
        {"name": "防御姿态", "effect": "defense*1.5", "duration": 3}
    ]
}
```

### 5.3 休闲/移动游戏数值设计

**移动游戏数值优化重点**：
1. **首次体验优化**：前5分钟决定留存率
2. **能量/体力系统**：控制游戏时长，鼓励回归
3. **进度成就感**：频繁的小目标达成
4. **IAP平衡**：免费体验完整，付费加速/美化

**F2P数值心理技巧**：
- **损失厌恶**：玩家更避免损失而非追求收益
- **沉没成本**：投入时间/资源后更不愿放弃
- **稀缺性原则**：限时/限量物品更有吸引力
- **社会证明**：看到他人拥有增加购买欲望

## 总结

数值设计是游戏开发中既需要科学严谨又需要艺术直觉的领域。记住这些核心原则：

1. **玩家体验优先**：数值应为游戏体验服务，而非相反
2. **平衡是过程**：数值平衡是持续迭代，而非一次性完成
3. **简单性价值**：简单的数值系统通常比复杂系统更好
4. **测试驱动**：让数据和玩家行为指导设计决策
5. **接受不完美**：追求"足够好"的平衡而非理论完美

**给独立开发者的实用建议**：

1. **从简单开始**：先用电子表格建模，再投入代码实现
2. **自动化测试**：编写脚本自动测试数值平衡
3. **收集玩家数据**：即使小规模测试也能提供宝贵见解
4. **版本控制数值**：数值调整应有记录和理由
5. **关注极端情况**：测试1级玩家和满级玩家的体验

最终，好的数值设计是隐形的。当玩家完全沉浸在游戏体验中，没有注意到数值的存在时，你就成功了。当玩家开始抱怨数值问题时，那就是需要迭代的信号。
